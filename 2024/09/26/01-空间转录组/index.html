空间转录组的流程分析

  1. 聚类
    • 不同的功能区域
      • 对比单细胞转录组分析的聚类：细胞类型
  2. 降维
    • 主要使用空间信息进行进行降维，如UMAP、t-SNE，同时考虑细胞之间的空间关系，但一般只有在多个切片联合分析才考虑降维
      • 对比单细胞转录组分析的降维：基于基因表达矩阵进行降维，通常使用PCA、t-SNE、UMAP等方法
  3. 差异分析
    • 主要发现不同空间区域之间差异表达的基因，如SpatiaIDE
      • 对比单细胞转录组分析的差异分析：用于发现不同细胞类型之间的差异表达基因，如Seurat的FindAllMarkers
  4. 富集分析
    • 主要用于解释在不同区域或细胞类型中富集的生物学过程，如GO分析、KEGG分析
      • 对比单细胞转录组分析的富集分析：用于解释在不同细胞类型中富集的生物学过程，如GO分析、KEGG分析
  5. 细胞类型
    • 相互糅合在一起，强调统一体
      • 对比单细胞转录组分析的细胞类型：批次分开，强调细胞类型的精确
  6. 肿瘤细胞区域
    • 图片可以直接识别
      • 对比单细胞转录组分析的肿瘤细胞区域：算法推断
  7. 细胞类型分布
    • 能够在组织中确定不同细胞类型的空间分布情况
      • 对比单细胞转录组分析的细胞类型分布：一个或几个cluster
  8. 共定位
    • 通常指的是组织或者细胞区域内同时表达的基因，这可以通过分析不同区域的基因表达矩阵来确定，共定位基因可能在特定的细胞类型或空间区域中表达，从而解释这些区域的生物学功能或发育过程
      • 对比单细胞转录组分析的共定位：通常指的是不同细胞类型之间共同表达的基因，这可以通过分析单细胞数据集中不同细胞簇的基因表达模式来确定，共定位基因可能多个细胞类型中表达，从而揭示这些细胞类型之间的功能联系或分析交互作用

DataFrame

pandas 库中的 DataFrame 是一种二维表结构，可以理解为一个类似于 Excel 表格或数据库表的对象，能够存储不同类型的数据。它的特点是带有行标签（index）和列标签（columns），便于数据的组织、操作和分析

scverse分析框架 - AnnData数据结构

等同于R中的S4对象，是数据结构

目前单细胞领域的工具分为三种：

  1. Bioconductor：R语言实现的生物信息学生态
  2. Seurat：R语言实现的单细胞分析生态
  3. Scverse：基于Python实现的单细胞分析生态
    • 单细胞主要用anndata
    • 单细胞多组学主要有mudata
    • 空间的多组学用SpatialData

SCVI-Tools(Single Cell Variational Inference Tools)：是一个开源的Python库，专为单细胞转录组数据的分析而设计

squidpy：专门用于处理和分析空间分子数据，基于scanpy和anndata，继承了它们的模块化和可拓展性，并专门处理带有空间坐标的数据提供独特的方法，包括整合组织图像进行深度分析

AnnData结构：

  1. 核心属性 X 为一个稀疏矩阵（scipy.sparse.csr.csr_matrix），用于存储矩阵的值
  2. 矩阵的行列分别对应观察者 obs（即细胞条形码），变量 var （即基因），这两个属性分别是一个数据框(panads.core.frame.DataFrame)类型用于存储行列的注释信息，其索引就是对应的 obs_names 和 var_names
功能 类型
adata.X 矩阵 numpy矩阵
adata.obs 观测量 pandas Dataframe
adata.var 特征量 pandas Dataframe
adata.uns 非结构化数据 字典dict

X数据槽：

  • X slot存储了当前用于分析的表达矩阵，一般使用CSR的稀疏矩阵格式
  • X 的一个数据结构是numpy array，是没有行名列名消息的，行名和列名消息存储在.obs和.var里

Layers数据槽：

  • Layers本质上是python的字典，每个layer对应一个表达矩阵的二位数组，比如标准化、对数化等等，这些转换后的数据可以存储在layer中

      • Comma-Separated Values（CSV）或文本文件：
        • .csv 或 .txt 文件，包含表达矩阵的文本文件。每一行代表一个基因，每一列代表一个细胞，元素是表达量

多个样本单细胞数据整合 - ingest和BBKNN

整合算法介绍

ingest

  • 一种基于PCA的简单方法用于集成数据（integrating data）
  • 用已知的细胞类型数据去注释未知的
  • ingest函数假设有一个带注释的参考数据集（有点类似SingleR），该数据集捕获了感兴趣的生物变异性。ingest是在参考数据上拟合模型，并使用它来投影新数据。目前，该模型是一个结合了邻居查找搜索树（neighbor lookup search tree）的PCA或者UMAP

ingest有以下特点：

  1. 由于ingest操作简单，过程清晰，因此工作流透明且快速
  2. 与BBKNN相似，ingest使数据矩阵本身保持不变。因为ingest是在embedding空间做分析
  3. 与BBKNN不同，ingest在解决了标签映射的同时，维持了可能具有特定簇或轨迹等所需属性的embedding（因为BBKNN是简单粗暴的对齐embedding）

BBKNN

  • BBKNN：Batch balanced KNN，BBKNN是一种快速直观的去除批次效应工具，可直接用于scanpy工作流。它是 scanpy.pp.neighbors() 的一种替代方法，这两个功能都是创建一个 neighbors graph，以便后续在聚类、pseudo time计算和UMAP可视化中使用
  • 标准方法首先确定整个数据集中每个细胞的k个最近邻，然后将候选细胞转换为指数相关连接，然后作为进一步分析的基础。如果数据中存在批次效应（可能是由于不同的数据采集技术、协议变更，甚至是特别严重的操作员影响），那么将很难在不同批次中链接相应的细胞类型
  • BBKNN修改了邻域构建步骤，以生成一个跨所有批次数据平衡的图。这种方法将邻居网络视为数据的主要表示形式。对于每个细胞，BBKNN图是通过独立地为每个设备提供的批次（单个数据集）中的每个细胞找到k个最近邻居来构建的，这使得每个细胞在每个批次中都有一个独立的邻居池。同类细胞的这些邻居都被加入列表联系起来
  • 注意：BBKNN会将列表中的元素用简单快速的方式对齐，即对齐不同批次的同类细胞。所以才说BBKNN实现了数据整合
  • BBKNN的主要假设是，批次间至少存在一些相同类型的细胞，并且批次效应引起的批次间相同细胞类型之间的差异小于批次内不同类型细胞之间的差异，在这种情况下，图的构造将跨批次将相似的细胞类型分组在一起，同时将不相关的细胞类型分开

Giotto

Giotto工具箱是Giotto软件包的升级版本，提供了处理、分析和可视化各种尺度和多种分辨率的空间多组学数据的工具。它的基本框架可推广到几乎所有当前和新兴的空间组学技术上，例如：原位杂交（seqFISH+, merFISH, osmFISH, CosMx）、测序（Slide-seq, Visium, STARmap, SeqScope, Stereo-Seq）以及基于成像的多组学技术（CyCIF, MIBI, CODEX）等。这些技术在分辨率（亚细胞、单细胞或多细胞）、空间维度（2D vs 3D）、分子形态（蛋白质、RNA、DNA等）和通量（细胞和分析物的数量）方面有所不同

R中单细胞空转联合分析 - RCTD

单细胞联合空间主流软件介绍

R中常用的软件：

  • Seurat：TransferAnchors
  • RCTD：robust cell type decomposition 鲁棒细胞类型分解
  • Spotlight：非负矩阵因子分解回归
  • CellTrek：训练多元随机森林（RF）模型，使用共享降维特征预测空间坐标
  • CARD：conditional autoregressive-based deconvolution，基于条件自回归的反卷积
  • Giotto：PAGE（Parametric Analysis of Gene Set Enrichment）
  • Giotto：HyperGeometric test 超几何检验（与MIA相似）
  • Giotto：rank，它不是基于基因集的方法，而是使用单细胞所有的表达数据
  • MIA：Multimodal intersection analysis（多模态交集分析）

python中常用的软件：

  • Scanorama
  • Tangram
  • Cell2location

RCTD

RCTD：一种利用从scRNA-seq中学习细胞类型的表达谱对空间转录组 spot 内混合细胞的类型进行稳健分解的工具 RCTD（robust cell type decomposition），并考虑到了来自不同测序方法数据间的系统误差，克服了平台效应（platform effect）的影响

教程：https://github.com/dmcable/spacexr

RCTD可以将单细胞类型或细胞类型混合分配到空间转录组spots上。RCTD 有三种模式：

  1. Doublet mode：它为每个Spot分配1-2种细胞类型，推荐用于高空间分辨率的技术，如Slide-seq和MERFISH
  2. Full mode：它为每个Spot分配任意数量的细胞类型，推荐用于空间分辨率较低的技术，如100微米分辨率的Visium
  3. Multi mode：doublet mode的扩展，可以在每个点上发现两种以上的细胞类型，作为full mode的替代选项

模式一：Doublet mode

R中单细胞空转联合分析 - CellTrek（了解）

CellTrek：可以结合单细胞和空间转录组数据来准确地定位组织内单个细胞的位置，并构建空间细胞图谱

  • 一种可以基于scRNA-seq和ST数据将单细胞直接映射回组织切片空间坐标的计算工具包。该方法不同于ST解卷积，能够更灵活和直接地研究具有空间地形的单细胞数据
  • CellTrek工具包还提供了两个下游分析模块，包括用于空间共定位分析的SColoc和用于空间共表达分析的SCoexp
