数组部分一些思路总结

  • 暴力求解：O(n^2)

  • 双指针法（定义快、慢指针）

    • 时间复杂度：O(n)
    • 通过快、慢指针在一个for循环能够完成两个for循环的工作
      • 快指针：寻找新数组的元素，新数组为不含目标元素的数组
      • 慢指针：指向更新新数组的下标位置

  • 双端指针

    • left = 0、right = nums.length - 1

  • 单侧快慢指针

    • 一个指针负责填充、另一个指针负责查找元素

  • 调用栈

    • Stack<T> stack = new Stack<>()
    • pop(x)
    • push()

  • 滑动窗口

    • 时间复杂度：O(n)
    • 窗口：满足其和大于等于s的长度最小的连续子数组
    • 窗口的起始位置移动规则：如果当前窗口大于s，窗口就要向前移动
    • 窗口的结束位置移动规则：窗口的结束位置为遍历数组的指针，也就是for循环的索引

判断链表是否有环

  1. 定义一个slow指针和一个fast指针
  2. slow指针每次移动一个结点，fast指针每次移动两个结点
  3. 如果fast和slow相遇，则一定有环

如何找到这个环的入口？

证明过程：

  • 如果相遇，slow指针走过结点数 x+y，fast指针走过结点数 x+y+n(y+z)。因为fast指针为一步走两个结点，slow指针为一步走一个结点，所以 2(x+y) = x+y+n(y+z)，进而得出 x = (n-1)(y+z)+z
  • 当n=1时，x=z，则定义在相遇结点一个指针index1，在头结点一个指针index2，如果index1 == index2，则为环的入口

三种哈希结构

  • map（HashMap）映射
    • 数组的大小是受限制的，而且如果元素很少，而哈希值太大会造成内存空间的浪费。
    • set是一个集合，里面放的元素只能是一个key

通过两个遍历序列确定一个二叉树

以中序遍历和后序遍历为例：

  • 第一步：如果数组大小为零的话，说明是空节点了。
  • 第二步：如果不为空，那么取后序数组最后一个元素作为节点元素。
  • 第三步：找到后序数组最后一个元素在中序数组的位置，作为切割点
  • 第四步：切割中序数组，切成中序左数组和中序右数组 （顺序别搞反了，一定是先切中序数组）
  • 第五步：切割后序数组，切成后序左数组和后序右数组
  • 第六步：递归处理左区间和右区间

  • 前序遍历和中序遍历构造二叉树

回溯算法

回溯算法定义

  • 回溯法也可以叫做回溯搜索法，它是一种搜索的方式
  • 回溯是递归的副产品，只要有递归就会有回溯
  • 回溯函数也就是递归函数，指的都是一个函数
  • 回溯的本质是穷举，穷举所有可能，然后选出我们想要的答案

回溯算法可以解决的问题

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 切割问题：一个字符串按一定规则有几种切割方式
  • 子集问题：一个N个数的集合里有多少符合条件的子集
  • 排列问题：N个数按一定规则全排列，有几种排列方式
  • 棋盘问题：N皇后，解数独等等

回溯算法的实质

  • 回溯法解决的问题都可以抽象为树形结构

  • 集合的大小就构成了树的宽度，递归的深度就构成了树的深度

贪心算法

  • 贪心的本质是选择每一阶段的局部最优，从而达到全局最优
  • 使用条件：
    • 刷题或者面试的时候，手动模拟一下感觉可以局部最优推出整体最优，而且想不到反例，那么就试一试贪心
  • 贪心算法的解题步骤：
    • 将问题分解为若干个子问题
    • 找出适合的贪心策略
    • 求解每一个子问题的最优解
    • 将局部最优解堆叠成全局最优解
  • 做题的时候，只要想清楚 局部最优 是什么，如果推导出全局最优，其实就够了
