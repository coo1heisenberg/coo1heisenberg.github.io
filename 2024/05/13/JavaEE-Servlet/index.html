JavaEE介绍

  • Java EE Enterprise Edition 企业开发 → 服务器开发
  • Servlet → Request\Response → 会话技术(Cookie/Session) → Web组件(Listener/Filter) → MVC

Servlet介绍

  • Server Applet缩写组成的，小程序
  • 运行在JavaEE容器（Tomcat）下的小程序
  • 通过Tomcat可以访问静态资源和动态资源，静态资源就是我们前端学的内容，比如HTML、JS、CSS、xml、字体文件、图片等内容，而Servlet提供的是动态资源的访问
  • Servlet的开发其实就是Servlet中的service方法的开发

Web应用的目录结构

  • Web应用的根目录（ROOT）
    • 可以直接访问的资源（主要是静态资源，比如html、js、css、图片等）
    • WEB-INF文件夹 （受保护的资源的文件：就是不能够直接访问的资源）
      • classes文件夹
        • classpath路径，放的是编译后的字节码文件和配置文件
      • lib文件夹
        • 放的是第三方的依赖（scope为compile、runtime的依赖）
      • web.xml
        • web应用的描述文件 → 基本上可以不使用了
      • 其他的文件（其他不想被直接访问的资源）

Maven工程的目录结构

  1. packaging的值为war，war包直接丢入到Tomcat的webapps就可以使用

  2. src\main新增一个webapp文件夹

  3. 在IDEA中识别为Maven工程

  4. Maven工程的目录结构

    Maven工程的根目录：

    • src文件夹

      • main文件夹（开发）

        • java文件夹（java代码）
        • resources文件夹（配置文件）
        • webapp文件夹（web资源-对应的是web资源目录）

      • test文件夹（测试）

      • pom.xml文件

      • target文件夹

        • classes文件夹
        • {artifactid}-{version}文件夹(WEB应用根目录)

※对应关系图（Maven工程与Web应用之间的对应关系）


颜色是一一对应的

常见的报错：

  • ClassNotFound：编译后的内容里没有这个类 → target/artifactid-version/WEB-INF/ 里的classes、lib目录
    • classes里没有 → src/main/java
    • lib里没有 → pom.xml
  • FileNotFound → target/artifactid-version/

在IDEA中开发Web应用

在idea中按照maven工程（web）的目录结构进行开发

需要我们做的事情：

  • packaging设置war（前面有提到），然后把Maven工程刷新一下
    • pom 父工程，提供子工程通用的内容，只有一个pom.xml，没有代码和文件
    • war web应用，要满足web应用的目录结构，要放在JavaEE容器下运行
    • jar 默认的打包方式，如果想要运行jar，里面要包含main方法
  • 增加Tomcat server
    • 右上角锤子旁边编辑配置
  • 提供deployment的配置（docBase）
    • artifact → war exploded
    • application context → 路径

@WebServlet注解源码

  • 通常使用的是value属性
    • 功能上和urlPatterns是一样的，但是value属性有一个好处是如果说该注解只使用了value属性的话，value= 可以省略不写

URL-Pattern

  1. 合法写法：

    • /XXX eg：/first、/hello1、/hello/world -> 最左边要有 / , 不可以省略

    • *.xxx eg: *.do、*.action，如果访问localhost:8080/demo3/hello/world.do访问 *.do

  2. 优先级：
    • 路径匹配（eg：/XXX） ＞ 类型匹配（eg：*.xxx）
    • 匹配度越高优先级越高
      • eg：/hello.do > *.do
  3. 默认Servlet（起着一个兜底的功能）
    • Tomcat有提供默认的Servlet，它的url-Pattern就是 /
    • 缺省匹配就是：没有找到与之对应的servlet的话，找的就是缺省servlet
  • Tomcat提供的默认的Servlet其实它做的事情就是：在你的Web应用的根目录下去找对应的资源（文件），如果能找到，就输出出来，如果找不到就404
  • 如果你自己也开发了一个Servlet，你的这个Servlet的URL-Pattern也是/ (@WebServlet("/"))，那么你这个Servlet就会变为默认的Servlet

Servlet的生命周期

方法名 时间 次数 说明
init() Servlet初始化 1 只执行一次，在service之前，用来用来做一些初始化
service() 访问Servlet对应的URL n 执行多次，用来处理业务
destroy() Servlet销毁 1 只执行一次，通常用来做资源的释放
  • Servlet的初始化是在@WebServlet注解的时候，有一个属性load-on-startup
    • 默认值是负数；
    • 如果为负数的话，意味着访问该servlet路径的时候，该Servlet才初始化；
    • 如果不为负数，则Tomcat启动的时候就初始化，并且会按照数字的顺序来初始化不同的Servlet
  • Servlet的销毁：
    • Tomcat停的时候

  • 执行顺序
    • LifeCycleServlet2's init
    • LifeCycleServlet3's init
    • http://localhost:8080/demo4/life1才会懒加载LifeCycleServlet1's init和LifeCycleServlet1's service
    • 访问谁谁执行LifeCycleServlet1's service/ LifeCycleServlet2's service
    • LifeCycleServlet1、2、3's destroy
